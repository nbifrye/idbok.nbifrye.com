# 第1章: デジタルアイデンティティ入門

本章では、デジタルアイデンティティに関する議論の出発点となる基本概念を整理します。
これから各章で扱う認証プロトコルやプライバシー技術を理解するための共通語彙を身につけることが目標です。

## デジタルアイデンティティとは

デジタルアイデンティティ（Digital Identity）とは、情報システム上で「誰か」や「何か」を表現する属性の集合です。
具体的には、ユーザー名、メールアドレス、組織、権限、デバイスIDなどの属性が結びついて、一つのアイデンティティを形成します。

物理世界における身分証明書や顔と同じく、デジタルアイデンティティもまたサービス提供者が利用者を区別し、適切な権限を付与するための基盤となります。

## 識別・認証・認可

デジタルアイデンティティを理解する上で最も重要な3つの概念が、識別（Identification）、認証（Authentication）、認可（Authorization）です。
これらはしばしば混同されますが、それぞれ異なる役割を持ちます。

- **識別（Identification）** — 「あなたは誰だと主張していますか？」という問いに答える。ユーザー名やメールアドレスを入力する行為がこれにあたる
- **認証（Authentication）** — 「本当にその人であることを証明できますか？」という問いに答える。パスワード入力、パスキー、生体認証などで本人性を確認する
- **認可（Authorization）** — 「そのリソースにアクセスする権限がありますか？」という問いに答える。認証された利用者に対して、具体的な操作権限を与える

たとえば Web アプリケーションでは、ユーザーがメールアドレスを入力（識別）し、パスワードやパスキーで本人確認を行い（認証）、その後ログイン済みユーザーとして許可された機能を利用する（認可）、という流れで使われます。

## アイデンティティに関わる登場人物

現代のデジタルアイデンティティ基盤は、複数の役割が連携して成り立っています。
ここでは最低限押さえておきたい登場人物を紹介します。

- **ユーザー（End User）** — サービスを利用する人。自分自身のアイデンティティを持つ主体
- **リライングパーティー（Relying Party, RP）** — ユーザーを受け入れる側のアプリケーションやサービス
- **アイデンティティプロバイダー（Identity Provider, IdP）** — ユーザーの認証情報を管理し、RPにアイデンティティを証明する役割
- **認可サーバー（Authorization Server）** — OAuth 2.0 などにおいて、アクセストークンを発行する役割

後の章で学ぶ OAuth 2.0 や OpenID Connect は、これらの登場人物の間でアイデンティティ情報を安全に受け渡すためのプロトコルです。

## なぜデジタルアイデンティティが重要なのか

インターネット上のほぼすべてのサービスは、利用者が誰であるかを前提に動作します。
アイデンティティ基盤が脆弱だと、なりすまし、情報漏洩、不正アクセスといった深刻なリスクに直結します。
逆に、適切に設計されたアイデンティティ基盤は、セキュリティとユーザー体験を両立させる強力な武器となります。

本書の後の章では、こうしたアイデンティティ基盤を支える技術とその設計思想を、実務に近い順番で学んでいきます。
