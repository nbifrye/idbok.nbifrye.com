# 第2章: 認証プロトコル

認証プロトコルは、ユーザーやシステムのデジタルアイデンティティを検証するための標準化された手順を定義します。
現代のWebアプリケーションやAPIにおいて、安全な認証と認可を実現するための基盤技術です。

## この知識領域で扱うトピック

- **OAuth 2.0** — APIアクセスの委任認可フレームワーク。認可コードフロー、PKCE、クライアントクレデンシャルなど複数のグラントタイプを提供する
- **OpenID Connect (OIDC)** — OAuth 2.0 の上に構築されたアイデンティティレイヤー。ID トークンによるユーザー認証を標準化する
- **SAML 2.0** — XMLベースのエンタープライズ向けシングルサインオンプロトコル。組織間のアイデンティティ連携に広く使われる
- **Kerberos** — チケットベースのネットワーク認証プロトコル。Active Directory環境の基盤
- **HTTP認証** — Basic、Digest、Bearer、相互TLSなどのHTTPレベルの認証メカニズム
